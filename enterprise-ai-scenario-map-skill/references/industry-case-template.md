# 行业AI应用案例输出模板

## 概览
本模板用于规范化输出同行业AI应用最佳实践案例，确保案例信息的完整性和可读性。

## 输出格式规范

### 整体结构
```markdown
# <行业名称> AI应用最佳实践

## 一、<核心场景1>
### <场景标题>
**技术实现路径**：<技术方案概述，包括核心技术、架构设计、关键组件>

**典型案例**：
| 落地企业 | 具体应用 | 技术提供商 | 实施效果 | 关键经验 |
|---------|---------|-----------|---------|---------|
| <企业名称> | <应用描述> | <供应商名称> | 1.<效果1><br>2.<效果2> | 1.<经验1><br>2.<经验2> |
| <企业名称> | <应用描述> | <供应商名称> | 1.<效果1><br>2.<效果2> | 1.<经验1><br>2.<经验2> |

## 二、<核心场景2>
...
```

## 字段填写说明

### 场景标题
- 格式：`<动词> + <具体任务>`
- 示例：
  - 自动化数据采集与工程量计算
  - 智能客服与问题解答
  - 营销文案自动生成
  - 供应链风险预测与预警

### 技术实现路径
- 内容要求：清晰描述技术方案，包括：
  - 核心技术栈（如：CV、NLP、LLM、知识图谱等）
  - 架构设计（如：前后端分离、微服务、云原生等）
  - 关键组件（如：OCR引擎、大模型、向量数据库等）
  - 数据流向（如：数据采集→清洗→处理→输出）
  - 集成方式（如：API对接、插件集成、独立部署等）

- 写作要点：
  - 使用专业术语，但避免过度技术化
  - 突出创新点和关键技术
  - 说明与现有系统的集成方式
  - 提及数据安全和隐私保护措施

- 示例：
```
技术实现路径：基于图像识别（CV）、BIM模型解析、自然语言处理（NLP）等技术，
实现图纸、合同等多源数据的自动化提取与算量。采用微服务架构，核心组件包括：
1. OCR引擎：提取CAD图纸中的文字信息
2. CV模型：识别构件类型和尺寸
3. NLP模块：解析合同条款和技术规范
4. 知识库：存储行业标准和规范
系统通过API与企业ERP系统集成，数据加密传输，确保信息安全。
```

### 典型案例表格

#### 列1：落地企业
- 格式：`<企业名称>（<国家/地区>）`
- 示例：
  - 青矩顾问（中国）
  - Skanska（瑞典）
  - 阿里巴巴（中国）

#### 列2：具体应用
- 内容要求：描述企业在该场景的具体应用情况
- 要素：
  - 应用范围（全公司/部分业务/试点项目）
  - 应用规模（覆盖多少用户/多少业务量）
  - 应用特色（与其他方案的差异）

- 示例：
  - 土建/安装全专业建模与算量
  - 基于BIM的实时工程量更新
  - 覆盖全国200+门店的智能客服

#### 列3：技术提供商
- 内容要求：
  - 如果是企业自研：标注"自研"
  - 如果是第三方：标注供应商名称和产品名称
  - 如果是合作：标注主要合作方

- 示例：
  - 自研青矩智慧造价机器人
  - Autodesk BIM 360 + AI插件
  - 百度智能云 + 腾讯云微搭

#### 列4：实施效果
- 内容要求：量化描述AI应用带来的效果
- 推荐指标类型：
  - 效率提升：效率提升X%、处理时间减少X分钟
  - 成本降低：成本降低X%、节省人力X人
  - 质量改进：准确率提升X%、错误率降低X%
  - 客户体验：客户满意度提升X%、响应时间缩短X%
  - 业务增长：营收增长X%、转化率提升X%

- 格式：
  - 使用`<br>`换行
  - 每个效果一行，使用序号
  - 尽可能量化，避免模糊描述

- 示例：
  ```
  1. 算量效率提升300%<br>
  2. 钢筋建模精度达98%<br>
  3. 人工干预减少80%
  ```

#### 列5：关键经验
- 内容要求：总结成功实施的关键经验
- 常见经验类型：
  - 流程标准化：建立标准流程、规范文档
  - 数据治理：数据清洗、数据标准化、数据质量监控
  - 技术选型：选择合适的技术栈、考虑扩展性
  - 组织保障：成立专项团队、高层支持
  - 持续优化：模型迭代、效果监控、反馈闭环

- 格式：
  - 使用`<br>`换行
  - 每个经验一行，使用序号
  - 简洁明了，突出要点

- 示例：
  ```
  1. 标准化图纸预处理流程
  2. 多线程并行作业架构
  3. 动态更新行业规范库
  ```

## 注意事项
1. 案例必须真实可信，来源明确（来自官方报道、权威媒体、行业报告等）
2. 实施效果必须有量化数据支撑，避免"效果显著"、"大幅提升"等模糊描述
3. 关键经验要有可操作性，避免空泛理论
4. 技术描述要准确，避免过度包装或技术术语错误
5. 每个场景至少提供2个典型案例，覆盖不同规模和类型的企业
6. 优先选择与中国企业或目标公司业务相似的案例
