## 第二层：推荐填写（基于第一层动态生成）

根据 Q1–Q4 的回答，动态生成 2–3 个追问，提供选项（不强制）。

**示例：根据职业推断的追问**

| 职业 | 追问 | 选项 |
|------|------|------|
| 培训师 | 你最常用哪些工具做课件？ | PPT / Canva / 视频剪辑 / 手写+拍照 |
| 产品经理 | 你的 PRD 通常是什么格式？ | Markdown / Word / 在线文档 / 原型工具 |
| 开发者 | 你主要用什么语言/框架？ | Python / JavaScript / Go / 其他 |
| 内容创作者 | 你主要发布在哪个平台？ | 公众号 / 知乎 / B站 / 抖音 |
| 销售/客户成功 | 你主要服务什么规模的客户？ | SMB / 中大客户 / 战略客户 |

**示例：根据服务对象推断的追问**

| 服务对象 | 追问 | 映射字段 |
|---------|------|---------|
| 企业管理层 | 你汇报时偏好什么风格？ | USER.md `$.汇报风格` |
| 一线员工 | 你希望学员培训后有什么变化？ | USER.md `$.培训目标` |
| 外部客户 | 你的方案通常需要几轮评审？ | USER.md `$.方案评审轮数` |

## 问卷流程控制

```
开始
  ↓
展示第一层（Q1-Q5）
  ↓
若 Q1-Q4 任一缺失 → 追问（不允许跳过）
  ↓
展示推断逻辑预览（reverse-engineer.md）
  ↓
用户确认推断 → 第二层追问（推荐，不强制）
  ↓
展示第三层（可选）
  ↓
汇总所有输入 → 进入配置生成
```

## 追问话术示例

- **信息不足**："我需要再了解一点——你说'主要输出物'，是指 Word 文档、PPT、还是代码？"
- **信息矛盾**："你提到职业是医生，但主要输出是代码片段——这两者的组合比较少见，帮我确认一下你的主要工作是什么？"
- **推荐追问**："基于你的回答，我有几个小问题（可选回答）："
