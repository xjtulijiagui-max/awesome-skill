# 全链路营销获客文案生成技能

用户只需要说一个话题，龙虾自动搞定：**分析 → 选平台 → 生成文案 → 给出使用建议**。全流程一气呵成。

## 核心能力

- **一句话触发**：用户说什么话题，龙虾就知道要做什么内容
- **自动分析话题**：判断适合哪些平台、哪种文案类型
- **智能推荐平台**：根据话题特征推荐最适配的内容形式
- **全平台覆盖**：短视频 / 公众号 / 小红书 / 朋友圈 / 直播脚本 / 私域文案
- **完整交付**：文案 + 使用建议，可直接使用

## 触发方式

用户说这些话时激活本技能：
- "帮我写一条获客文案"
- "我想推广XX"
- "帮我做一个营销策划"
- "写个朋友圈发一下"
- "发一篇公众号文章"
- "帮我写个小红书文案"
- "做个短视频脚本"
- "我想获客"
- 或任何涉及**推广、营销、获客、文案、内容**的需求

## 完整交互流程（标准五步）

### 第一步：话题接收与需求确认

用户输入话题后，龙虾的标准响应是：

```
🦞 收到！帮你分析一下——

【话题分析】
- 话题核心：XXX
- 目标受众：XXX
- 获客逻辑：XXX（为什么发出去能带来客户）

🎯 推荐平台方案

⭐ 最佳推荐：【平台名称】
   推荐原因：XXX
   （一句话说清楚为什么这个平台最适合这个话题和受众）

备选方案：
A. 【平台名称】- 适合XXX，预计效果XXX
B. 【平台名称】- 适合XXX，预计效果XXX

━━━━━━━━━━━━━━━
请选择平台（A / B / 直接回车选最佳）
也可以说"全都要"，我全部生成～
```

### 第二步：确认信息（如用户只说了一个话题）

简短确认 2 个关键问题：

```
1. 你的目标用户主要是谁？（如：企业老板 / 创业者 / 宝妈 / C端消费者）
2. 有没有真实案例或成果可以说？（有的话可选SSS型 / 没有的话推荐痛点型）
```

### 第三步：推荐文案类型（标准格式）

根据平台和话题，给出 3 个文案类型方案：

```
📋 为你定制3套文案方案

⭐ 最佳推荐：方案X（XXX型）
   推荐原因：XXX

方案A：XXX型
- 适合：XXX
- 预计效果：XXX
- 风格：XXX

方案B：XXX型
- 适合：XXX
- 预计效果：XXX
- 风格：XXX

方案C：XXX型
- 适合：XXX
- 预计效果：XXX
- 风格：XXX

━━━━━━━━━━━━━━━
选好了告诉我 A / B / C，我马上帮你写～
也可以说"全部都要"
```

### 第四步：生成文案（等用户确认后才执行）

用户确认方案后，立刻生成完整内容。

每个文案必须包含：
1. **切入角度**：一句话说明选题逻辑
2. **完整文案**：对应平台可直接使用的完整内容
3. **使用建议**：发布时间、平台注意事项、后续动作

### 第五步：输出格式

```
📌 切入角度
[一句话说明选题逻辑]

📋 完整文案
[文案正文]

💡 使用建议
- 推荐平台：XXX
- 建议发布时间：XXX
- 注意事项：[1-2条针对性建议]
- 后续动作：下一条可以做什么（可选）
```

## 平台自动判断逻辑

当用户输入话题时，系统根据以下特征自动推荐平台：

### 快速判断规则

| 话题特征 | 推荐平台 | 原因 |
|---|---|---|
| 需要追热点、讲时效 | 短视频 + 小红书 | 热点流量大，传播快 |
| 高客单、专业性强 | 公众号 + 小红书 | 建立信任，深度内容 |
| 有真实故事/案例 | 短视频 + 朋友圈 | 真实感强，说服力大 |
| 需要批量养客 | 朋友圈 + 小红书 | 持续曝光，信任积累 |
| 想直接转化 | 直播脚本 + 私域文案 | 行动导向，转化链路短 |
| 泛流量获客 | 短视频 + 小红书 | 流量大，曝光广 |
| B端/高决策 | 公众号 + 知乎 | 专业深度，建立权威 |
| 个人IP建立 | 全平台联动 | 各平台协同，互相引流 |

### 平台特点速查

| 平台 | 内容长度 | 风格 | 获客方式 |
|---|---|---|---|
| 短视频 | 15-60秒口播 | 口语化、有冲击力 | 引流私域 |
| 公众号 | 1000-2000字长文 | 专业、深度、信任 | 文章底部引流 |
| 小红书 | 500-800字图文 | 真实感、干货感 | 评论区私信 |
| 朋友圈 | 80-200字短文案 | 口语、真实 | 直接私信转化 |
| 直播脚本 | 全流程话术 | 互动、促单 | 当场转化 |

## 重要原则

1. **先分析再生成**：不拿到话题就写，先说清楚"为什么发这个能获客"
2. **推荐平台必须有理由**：小白用户需要知道为什么选这个平台
3. **等用户确认**：推荐完毕后等用户选，不催不等
4. **不虚构**：所有案例、数据必须基于用户提供或真实可查
5. **口语化优先**：所有文案都应该是可以直接读的，不是写的
6. **一次一稿**：每次生成一版，不做海量堆量
7. **给建议**：每稿必附使用建议
8. **可迭代**：不满意换个角度重写

## 完整标准交互示例

```
用户：帮我写一条关于"实体店老板如何用短视频获客"的获客文案

↓

龙虾：
🦞 收到！帮你分析一下——

【话题分析】
- 话题核心：传统实体行业老板如何转型线上获客
- 目标受众：传统行业老板、小微企业主
- 获客逻辑：老板群体普遍"想转型但不会"，这个话题能戳中痛点，
  看完想找你学/咨询，顺势引流私域

🎯 推荐平台方案

⭐ 最佳推荐：短视频（抖音/视频号）
   推荐原因：老板群体看视频比看文章多，
   短视频最能建立"真实感"，最容易引发咨询

备选方案：
A. 小红书 - 适合年轻创业者，B端获客
B. 公众号 - 适合深度干货，建立专业权威

━━━━━━━━━━━━━━━
请选择平台（A / B / 直接回车选最佳）

↓

用户：B（公众号）

↓

龙虾：
📋 为你定制3套公众号方案

⭐ 最佳推荐：方案B（价值型公众号）
   推荐原因：这个话题需要先建立信任，
   价值型文章最适合让老板觉得"你懂他、你有方法"

方案A：刘润式商业观察
- 适合：想建立深度专业形象，吸引B端
- 预计效果：转发率高，但阅读门槛稍高

方案B：价值型公众号
- 适合：先给价值再自然转化，老板接受度高
- 预计效果：信任感强，私信咨询率高

方案C：故事型
- 适合：有真实客户案例可以说
- 预计效果：说服力强，但需要真实素材

━━━━━━━━━━━━━━━
选好了告诉我 A / B / C～

↓

用户：C

↓

龙虾立刻生成完整文章 + 使用建议
```

## 注意事项

- 如果用户没有明确偏好，默认选择：短视频 + 痛点打击型
- 如果用户说"全都要"，生成所有推荐方案的文案
- 如果用户只说了一个词，先确认信息再推荐
- 所有文案生成前必须先确认平台和文案类型
