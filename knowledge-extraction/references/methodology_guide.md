# 企业知识萃取方法论指南

## 本指南说明

本指南提供企业知识萃取的方法论框架选择、最佳实践和常见问题解答，帮助萃取工作更高效、更标准。

---

## 一、方法论框架对比

### 1.1 杨三角框架（杨国安教授组织能力模型）

**核心思想：** 组织能力 = 员工思维（愿）+ 员工能力（会）+ 员工治理（允）

**萃取维度：**
- **战略层面（为什么要干）**：外部趋势、起点状态、战略选择
- **意愿层面（要不要干）**：干部意愿、团队共识、激励机制
- **方法与资源层面（怎么干）**：方法论、关键动作、能力建设、资源配套
- **成效与证据**：数据化成果、投入产出、可复制性

**适用场景：**
- ✅ 大型企业数字化转型项目
- ✅ 涉及组织变革的AI项目
- ✅ 需要系统性方法论的战略级项目
- ✅ 3-5年的长期转型规划
- ✅ 材料包含战略规划、组织变革、文化建设

**优势：**
- 系统性强，覆盖战略到执行的完整链路
- 关注组织和人的因素，不只是技术
- 适合提炼组织能力和变革管理经验
- 适合高层汇报和战略传播

**局限：**
- 对具体业务场景的深度可能不足
- 如果材料不丰富，难以支撑四个维度
- 需要对组织变革有深刻理解

### 1.2 STAR2+5W2H框架

**核心思想：** 从具体场景出发，还原完整过程，提炼可复制经验

**萃取维度：**
- **场景识别（Scene）**：业务背景、问题识别（5W2H）、场景选择
- **过程还原（Task & Action）**：目标设定、行动实施、协作机制
- **结果评估（Result）**：目标达成、数据化成果、典型案例
- **反思提炼（Reflection）**：成功/失败原因、底层逻辑
- **复制推广（Replication）**：可复制性、适用场景、复制指南

**适用场景：**
- ✅ 具体AI业务案例萃取
- ✅ 单一场景或项目的深度分析
- ✅ 业务场景的方法论沉淀
- ✅ 可复制的业务模式总结
- ✅ 材料主要是具体项目、业务场景

**优势：**
- 聚焦具体场景，分析更深入
- 过程还原详细，可操作性强
- 适合提炼业务落地的实战经验
- 容易理解和传播

**局限：**
- 战略层面的视野可能不够
- 如果项目太复杂，单一场景可能不够全面
- 需要丰富的项目细节材料

### 1.3 混合框架（战略+场景双视角）

**核心思想：** 综合杨三角和STAR2的优势，既关注战略，也关注场景

**萃取维度：**
- **战略层面（杨三角框架）**：为什么要干、要不要干、怎么干、成效
- **场景层面（STAR2框架）**：选取3-5个典型场景深度萃取
- **整合分析**：战略与场景的互动、跨场景共性提炼

**适用场景：**
- ✅ 复杂的大型AI转型项目
- ✅ 需要同时覆盖战略和场景的萃取任务
- ✅ 包含多个业务场景的综合萃取
- ✅ 需要全面视角的标杆案例分析
- ✅ 材料既有战略文档又有业务案例

**优势：**
- 全面覆盖，既有战略高度又有场景深度
- 适合复杂的、多场景的项目
- 可以展示战略与场景的互动关系
- 最适合大型标杆案例分析

**局限：**
- 工作量较大，需要丰富的材料
- 可能内容过多，不够聚焦
- 需要较强的整合分析能力

## 二、框架选择决策树

### 2.1 快速选择流程图

```
开始
│
├─ 第一步：判断项目类型
│   ├─ 组织变革类项目 → 进入第二步
│   ├─ 具体业务案例 → 进入第三步
│   └─ 综合类项目 → 进入第四步
│
├─ 第二步：组织变革项目
│   ├─ 材料是否丰富？
│   │   ├─ 是 → 杨三角框架
│   │   └─ 否 → STAR2框架（聚焦关键场景）
│   │
│   └─ 萃取范围？
│       ├─ 战略级 → 杨三角框架
│       └─ 业务级 → STAR2框架
│
├─ 第三步：具体业务案例
│   ├─ 是否只有一个场景？
│   │   ├─ 是 → STAR2框架
│   │   └─ 否 → 混合框架（多场景）
│   │
│   └─ 材料充实度？
│       ├─ 丰富 → 混合框架
│       └─ 稀少 → STAR2框架
│
└─ 第四步：综合类项目
    ├─ 战略和场景哪个更重要？
    │   ├─ 战略更重要 → 杨三角框架
    │   ├─ 场景更重要 → STAR2框架
    │   └─ 都重要 → 混合框架
    │
    └─ 时间是否充裕？
        ├─ 是 → 混合框架
        └─ 否 → 杨三角或STAR2（选主次）
```

### 2.2 判断标准清单

**选择杨三角框架的判断标准（满足3项以上）：**
- [ ] 项目涉及组织架构调整
- [ ] 项目涉及文化建设或变革管理
- [ ] 项目周期较长（3年以上）
- [ ] 材料包含战略规划、组织变革、人才培养
- [ ] 目标受众是高层管理者
- [ ] 需要提炼组织能力而非业务经验

**选择STAR2框架的判断标准（满足3项以上）：**
- [ ] 项目聚焦具体业务场景
- [ ] 材料主要是项目文档、访谈记录
- [ ] 需要还原详细的实施过程
- [ ] 目标受众是业务执行者
- [ ] 需要提炼可复制的业务经验
- [ ] 时间有限，需要快速产出

**选择混合框架的判断标准（满足3项以上）：**
- [ ] 项目复杂，包含多个层面和场景
- [ ] 材料丰富，既有战略文档又有业务案例
- [ ] 需要全面展示项目的各个方面
- [ ] 时间充裕，可以深度挖掘
- [ ] 目标受众多样（高层+执行层）
- [ ] 需要展示战略与场景的互动关系

## 三、萃取最佳实践

### 3.1 文档处理最佳实践

**批量转换优先：**
```
# 错误做法：逐个读取Word文档
read(docx_file1)  # 慢
read(docx_file2)  # 慢
read(docx_file3)  # 慢

# 正确做法：批量转换为MD，再读取
python doc_converter.py "源目录" -o "converted/"
read(md_file1)  # 快
read(md_file2)  # 快
read(md_file3)  # 快
```

**增量处理：**
- 使用转换脚本的日志功能
- 只转换新增或修改的文件
- 避免重复处理，提高效率

**并行读取：**
- 同时读取多个MD文件
- 使用Glob和Grep快速定位信息
- 优先阅读标题和摘要

### 3.2 访谈提纲设计最佳实践

**模板化定制：**
1. 先读取标准模板
2. 根据萃取对象增加针对性问题
3. 根据行业特点增加专业问题
4. 根据材料缺口补充问题

**模块化组织：**
- 现状调研模块
- 需求挖掘模块
- 资源评估模块
- 实施规划模块

**量化数据优先：**
- 设计能获得量化数据的问题
- 避免是/否的简单问题
- 鼓励举例说明

**案例深挖：**
- 要求具体案例
- 追问关键决策
- 了解失败教训

### 3.3 萃取报告生成最佳实践

**数据支撑充分：**
- 每个观点都要有数据或案例支撑
- 避免空泛描述
- 用图表可视化数据

**方法论清晰：**
- 提炼的方法论要可复制
- 提供具体步骤和Checklist
- 说明适用条件和限制

**结构标准化：**
- 严格按照模板结构组织
- 保持逻辑一致性
- 使用统一的术语

**可复制性：**
- 重点提炼可复制的经验
- 标注不可复制的特例
- 提供复制指南

### 3.4 框架使用技巧

**杨三角框架使用技巧：**
1. **战略层面**：聚焦外部驱动和内部差距
2. **意愿层面**：关注人的因素（干部、团队、激励）
3. **方法层面**：提炼关键动作（8-12个标准化动作）
4. **成效层面**：量化数据+ROI+可复制性

**STAR2框架使用技巧：**
1. **场景识别**：用5W2H全面分析问题
2. **过程还原**：详细记录阶段和关键动作
3. **结果评估**：数据对比+典型案例
4. **反思提炼**：从表象到底层逻辑
5. **复制推广**：提供具体Checklist

**混合框架使用技巧：**
1. **战略部分**：用杨三角，但聚焦关键点
2. **场景部分**：选3-5个典型场景，用STAR2深度萃取
3. **整合分析**：提炼战略与场景的互动关系
4. **控制篇幅**：避免内容过多，保持聚焦

## 四、常见问题与解答

### Q1: 材料不够丰富，怎么办？

**问题表现：**
- 只有访谈记录，缺少文档
- 文档内容简单，深度不够
- 缺少量化数据

**解决方案：**
1. **选择合适的框架**：材料少时，选择STAR2框架（更聚焦）
2. **深挖现有材料**：从有限材料中提取所有可用信息
3. **标注信息缺口**：在报告中明确标注"待补充"
4. **基于访谈补充**：通过访谈补充缺失信息
5. **调整萃取深度**：聚焦最核心的1-2个点，不求全面

### Q2: 不知道选择哪个框架？

**问题表现：**
- 项目特点不明确
- 材料类型多样
- 用户需求模糊

**解决方案：**
1. **问用户3个问题**：
   - 这个项目是偏战略还是偏业务？
   - 最想萃取什么？（组织能力 vs 业务经验）
   - 目标受众是谁？（高层 vs 执行层）
2. **看材料类型**：战略文档多→杨三角，业务案例多→STAR2
3. **用决策树**：按照框架选择决策树逐步判断
4. **不确定就用混合框架**：覆盖更全面，但工作量更大

### Q3: 访谈提纲设计太耗时？

**问题表现：**
- 每次都要从头设计
- 问题质量不稳定
- 缺少标准化

**解决方案：**
1. **使用标准模板**：直接使用skill提供的4类访谈模板
2. **快速定制**：在标准模板基础上增加10%的针对性问题
3. **复用旧提纲**：类似项目可以复用80%的问题
4. **模板库**：建立常见场景的提纲模板库

### Q4: 萃取报告质量不高？

**问题表现：**
- 内容空泛，缺少深度
- 数据支撑不足
- 方法论不清晰

**解决方案：**
1. **数据优先**：每个观点都要有数据或案例支撑
2. **深挖案例**：每个要点至少配1个具体案例
3. **提炼方法论**：不仅描述"做了什么"，更要提炼"为什么这样做"
4. **对照模板检查**：严格按照模板结构检查是否完整
5. **请专家评审**：请领域专家评审报告质量

### Q5: 文档处理效率低？

**问题表现：**
- 逐一读取文档太慢
- Word文档无法直接读取
- 重复处理浪费时间

**解决方案：**
1. **必须批量转换**：使用doc_converter.py一次性转换所有文档
2. **增量处理**：利用转换日志，只处理新增/修改文件
3. **并行读取**：同时读取多个MD文件
4. **优先级排序**：先读重要文档，再读辅助文档
5. **使用搜索**：用Grep搜索关键词，快速定位信息

## 五、萃取工作流程SOP

### 5.1 完整萃取流程

**阶段1：准备（1-2天）**
1. 明确萃取目标和范围
2. 收集材料（文档、访谈记录、数据）
3. 确定萃取类型（访谈提纲 vs 萃取报告）
4. 选择方法论框架

**阶段2：文档处理（1-2天）**
1. 批量转换文档（DOCX/XLSX/PPTX → MD）
2. 快速浏览所有材料
3. 按重要性和相关性排序
4. 识别关键信息和缺口

**阶段3：访谈/分析（3-5天）**
1. 如果是访谈提纲：设计针对性问题
2. 如果是萃取报告：深度分析材料
3. 提取关键信息（数据、案例、方法论）
4. 填充到模板结构中

**阶段4：初稿撰写（2-3天）**
1. 按照模板结构撰写
2. 数据支撑充分
3. 案例丰富
4. 方法论清晰

**阶段5：评审修订（1-2天）**
1. 内部评审（完整性、准确性）
2. 专家评审（方法论、深度）
3. 用户反馈（实用性）
4. 最终修订

**总计：8-14天**（根据材料丰富度和萃取深度）

### 6.2 快速萃取流程（适用于材料丰富、时间紧迫）

**Day 1：准备+文档处理**
- 上午：明确目标、收集材料
- 下午：批量转换文档、快速浏览

**Day 2：深度分析**
- 上午：提取关键信息（数据、案例）
- 下午：选择框架、开始填充

**Day 3：撰写+修订**
- 上午：完成初稿
- 下午：内部评审、修订

**总计：3天**

## 七、持续改进

### 7.1 萃取质量评估

**访谈提纲质量评估：**
- [ ] 问题是否有针对性（非通用模板）
- [ ] 是否能获得量化数据
- [ ] 是否覆盖关键模块
- [ ] 是否有案例深挖问题

**萃取报告质量评估：**
- [ ] 数据支撑是否充分
- [ ] 案例是否丰富
- [ ] 方法论是否清晰可复制
- [ ] 结构是否标准化
- [ ] 逻辑是否清晰

### 8.2 用户反馈收集

**每次萃取后收集反馈：**
1. 哪些部分最有价值？
2. 哪些部分需要改进？
3. 是否有遗漏的内容？
4. 框架选择是否合适？
5. 整体满意度如何？

### 8.3 持续优化

**基于反馈优化：**
1. 更新模板（增加新模块、优化问题）
2. 完善方法论（提炼新方法、总结新规律）
3. 补充案例（增加典型案例库）
4. 优化流程（提高效率、减少重复）
