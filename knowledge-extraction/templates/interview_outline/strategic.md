# 战略层访谈提纲模板

## 访谈对象说明

**典型岗位：**
- 高管层（CEO、CTO、CDO、CIO等）
- 战略部门负责人
- 投资方/董事会代表
- 集团科技管理部门负责人

**项目经历要求：**
- 参与过企业AI战略制定或数字化转型规划
- 对AI技术投资有决策权或建议权
- 了解企业3-5年的数字化发展路线图

**每场时长：** 60-90分钟

**建议人数：** 1-2人，覆盖战略制定和战略执行两个视角

## 访谈核心目标

萃取企业在AI战略层面的顶层设计和决策逻辑：
- 了解AI战略的制定背景、核心目标和投资布局
- 把握数字化转型的战略方向、优先级和阶段目标
- 了解高层对AI价值的期望、考核标准和成功定义
- 识别AI战略与业务战略的融合机制
- 提炼可复制的战略制定方法论

## 核心访谈问题

### 模块一：战略洞察与外部趋势

**问题背景：** 了解企业制定AI战略的外部环境和行业背景

**核心问题：**
1. 当前行业在AI应用方面的主要趋势和变化是什么？
2. 竞争对手在AI领域的布局和投入情况如何？
3. 客户或市场对AI技术的接受度和需求变化如何？
4. 政策法规对企业AI应用有哪些影响或要求？
5. 如果不进行AI转型，企业面临的最大风险是什么？

**预期产出：**
- 外部环境分析（行业趋势、竞争态势、客户需求、政策环境）
- "不改变的风险"清单
- 外部压力驱动因素

### 模块二：战略目标与愿景

**问题背景：** 了解企业AI战略的核心目标和愿景定义

**核心问题：**
1. 企业AI战略的核心愿景是什么？（用一句话描述）
2. AI战略支撑的业务目标有哪些？（如收入增长、成本降低、效率提升、客户体验优化等）
3. 未来3-5年的AI战略里程碑是什么？
4. AI投资的预期回报是什么？（财务ROI、战略价值）
5. 如何定义AI战略的成功？（关键指标和评价标准）

**预期产出：**
- AI战略愿景陈述
- 战略目标清单（业务价值维度）
- 阶段性里程碑路线图
- 成功标准和KPI体系

### 模块三：战略选择与决策逻辑

**问题背景：** 了解AI战略制定过程中的关键决策和取舍

**核心问题：**
1. 为什么选择当前的AI战略方向？（与其他可选方案的对比）
2. AI战略的边界在哪里？（做什么、不做什么）
3. 在战略制定过程中，最重要的3个决策是什么？
4. 哪些业务场景优先考虑？选择标准是什么？
5. 技术路线的选型依据是什么？（如自研vs外购、大模型vs小模型）
6. 如果重来一次，哪些决策会做出不同的选择？

**预期产出：**
- 战略决策逻辑和取舍原则
- 优先级评估框架
- 技术路线选择方法论
- 关键决策案例

### 模块四：战略承接与组织保障

**问题背景：** 了解AI战略如何在组织内落地和执行

**核心问题：**
1. AI战略是如何分解到各业务部门和职能部门的？
2. 各层组织对AI战略的理解和认同程度如何？
3. 有哪些机制确保AI战略的有效执行？（如季度复盘、专项项目、考核激励）
4. 战略执行过程中遇到的最大阻力是什么？如何克服？
5. 如何平衡短期收益和长期投入？
6. AI战略的调整机制是什么？何时调整、如何调整？

**预期产出：**
- 战略分解机制
- 组织共识建立方法
- 战略执行保障机制
- 变革管理和阻力应对

### 模块五：资源配置与投资管理

**问题背景：** 了解AI战略的资源投入和投资管理

**核心问题：**
1. AI战略的预算规模和分配原则是什么？
2. 人才、技术、数据三类资源的投入重点是什么？
3. 如何评估AI项目的投资回报？
4. 哪些资源是内部自建，哪些依赖外部合作？
5. 资源投入的节奏如何规划？（分阶段投入策略）

**预期产出：**
- 资源配置框架（资金、人才、技术、数据）
- ROI评估方法
- 自建vs外购决策逻辑
- 分阶段投入策略

### 模块六：战略成效与经验总结

**问题背景：** 提炼战略制定和执行的经验教训

**核心问题：**
1. AI战略执行以来，最显著的成效是什么？（数据支撑）
2. 哪些预期尚未达成？原因是什么？
3. 战略制定过程中最值得分享的经验是什么？
4. 最大的失误或遗憾是什么？如果能重来会如何改进？
5. 对其他企业制定AI战略的建议是什么？

**预期产出：**
- 战略成效数据（量化成果）
- 成功经验提炼
- 失败教训总结
- 可复用的战略制定方法论

## 访谈注意事项

1. **聚焦决策逻辑：** 不仅了解"做了什么决策"，更要理解"为什么这样决策"
2. **量化数据优先：** 尽可能收集具体数字（预算金额、ROI指标、时间节点）
3. **案例深度挖掘：** 对关键决策要深入了解背景、选项、权衡、结果
4. **保留原始表述：** 记录访谈对象的直接引语，特别是有洞察力的观点
5. **追问矛盾点：** 对表述中的矛盾或不一致之处要深入追问

## 输出格式建议

访谈结束后，整理为以下结构：
1. 访谈基本信息（时间、对象、职位）
2. 访谈纪要（按问题模块组织）
3. 关键发现（3-5个核心洞察）
4. 数据清单（重要数据和指标）
5. 待验证问题（需与其他访谈对象交叉验证的问题）
