# IT产品经理访谈提纲模板

## 访谈对象说明

**典型岗位：**
- AI产品经理
- 数字化产品经理
- 智能化业务产品经理
- 技术架构师
- 数据产品经理

**项目经历要求：**
- 主导过1个以上AI产品的端到端落地（需求分析→方案设计→上线推广）
- 熟悉AI技术栈（大模型、RAG、Agent、提示词工程等）
- 有业务需求转化为技术方案的经验
- 了解产品上线后的运营和迭代

**每场时长：** 60-90分钟

**建议人数：** 2-3人，覆盖前端和后端产品经理

## 访谈核心目标

萃取AI产品从0到1的完整落地方法论：
- 了解AI产品的需求发现和场景定义方法
- 收集技术方案选型和设计逻辑
- 提炼产品落地和上线推广的经验
- 识别AI产品落地中的关键成功因素和常见陷阱
- 总结可复制的AI产品开发流程

## 核心访谈问题

### 模块一：需求发现与场景定义

**问题背景：** 了解AI产品经理如何识别和定义AI场景

**核心问题：**
1. AI产品需求的主要来源是什么？（业务反馈、数据分析、竞品研究、技术探索）
2. 如何验证一个AI需求的真实性和价值？
3. 场景定义的标准流程是什么？
4. 如何判断一个场景适合用AI解决？
5. 需求优先级如何排序？（价值、可行性、成本）
6. 有哪些需求被否决了？为什么？

**预期产出：**
- 需求发现渠道和方法
- 需求验证框架
- 场景定义流程
- AI适用性判断标准
- 优先级评估模型

### 模块二：技术方案选型与设计

**问题背景：** 了解AI产品的技术方案选择和设计逻辑

**核心问题：**
1. 技术方案选型的决策依据是什么？（性能、成本、复杂度、可维护性）
2. 大模型 vs 小模型：什么场景下选择哪种？为什么？
3. RAG技术的适用场景是什么？如何设计知识库？
4. 何时使用Agent？如何设计Agent的能力边界？
5. 提示词工程是如何开展的？有哪些优化方法？
6. 如何平衡技术先进性和落地可行性？
7. 有哪些技术选型的教训？

**预期产出：**
- 技术选型决策树
- 大小模型选择标准
- RAG设计方法论
- Agent设计原则
- 提示词工程最佳实践
- 技术选型避坑指南

### 模块三：产品设计与用户体验

**问题背景：** 了解AI产品的设计理念和用户体验考虑

**核心问题：**
1. AI产品的设计原则是什么？（如简单性、透明性、可控性）
2. 如何设计AI的人机交互界面？
3. 如何让用户理解AI的能力和限制？
4. 如何设计AI的反馈机制？（如置信度、解释性）
5. 如何处理AI的错误或不确定性？
6. 如何平衡自动化和人工干预？
7. 用户体验测试是如何开展的？

**预期产出：**
- AI产品设计原则
- 人机交互设计模式
- AI能力表达方法
- 错误处理机制
- 人工干预设计
- UX测试方法

### 模块四：产品开发与实施

**问题背景：** 了解AI产品的开发流程和实施方法

**核心问题：**
1. AI产品的开发流程是什么？（与普通产品的差异）
2. 数据准备和模型训练的周期是多长？
3. 如何保证AI产品的质量和准确性？
4. 测试策略是什么？（功能测试、性能测试、效果评估）
5. 上线前的灰度策略是什么？
6. 如何处理模型迭代和版本管理？
7. 开发过程中最大的技术挑战是什么？

**预期产出：**
- AI产品开发流程
- 数据和模型管理方法
- 质量保证机制
- 测试和灰度策略
- 模型迭代流程
- 技术挑战应对方案

### 模块五：产品上线与推广

**问题背景：** 了解AI产品的上线策略和推广方法

**核心问题：**
1. 产品上线后的推广策略是什么？
2. 如何收集用户反馈和数据？
3. 运营指标有哪些？如何监控？
4. 如何进行产品迭代？迭代周期是多久？
5. 用户培训是如何开展的？
6. 如何应对用户的质疑或抵触？
7. 上线后遇到的最大的意外是什么？

**预期产出：**
- 产品推广策略
- 用户反馈机制
- 运营指标体系
- 产品迭代方法
- 用户培训方案
- 风险应对案例

### 模块六：业务-技术协同

**问题背景：** 从产品视角了解业务与技术团队的协作

**核心问题：**
1. 产品经理如何理解业务需求？
2. 需求是如何传递给开发团队的？
3. 如何确保技术方案满足业务需求？
4. 跨部门协作中有哪些常见问题？如何解决？
5. 产品经理在其中扮演什么角色？（桥梁、翻译、决策者）
6. 有哪些协同工具或机制？

**预期产出：**
- 需求理解和管理方法
- 需求传递流程
- 方案验证机制
- 跨部门协作模式
- 产品经理角色定位
- 协同工具和机制

### 模块七：知识沉淀与团队协作

**问题背景：** 了解AI相关的知识管理和团队协作

**核心问题：**
1. 知识库是如何建设和维护的？
2. 业务知识如何转化为AI可理解的形式？
3. 提示词是如何管理的？（版本、复用、优化）
4. 团队内部的知识分享机制是什么？
5. 如何培养团队的AI能力？
6. 有哪些知识管理工具或平台？

**预期产出：**
- 知识库建设方法论
- 知识转化流程
- 提示词管理体系
- 知识分享机制
- 团队能力培养方法
- 知识管理工具清单

### 模块八：经验总结与最佳实践

**问题背景：** 提炼可复制的经验和避坑指南

**核心问题：**
1. AI产品成功的关键因素是什么？（排序）
2. 最大的坑或教训是什么？
3. 如果重来一次，哪些地方会做得不一样？
4. 哪些经验可以复制到其他AI产品？
5. 对新人有什么建议？
6. 有哪些最佳实践可以分享？

**预期产出：**
- 关键成功因素（按重要性排序）
- 常见陷阱清单
- 改进建议
- 可复制经验
- 新人建议
- 最佳实践案例

## 访谈注意事项

1. **聚焦方法论：** 不仅了解"做了什么"，更要理解"为什么这样做"和"如何做"
2. **关注技术细节：** AI产品的技术细节很重要，要深入了解
3. **收集数据指标：** 性能、准确率、用户满意度等要有具体数字
4. **深挖失败案例：** 失败案例往往比成功案例更有价值
5. **识别可复用组件：** 如提示词模板、知识库结构、设计模式等

## 输出格式建议

访谈结束后，整理为以下结构：
1. 访谈基本信息
2. 产品背景（目标、用户、场景）
3. 需求发现和场景定义方法
4. 技术方案选型逻辑
5. 产品设计和用户体验方案
6. 开发和实施流程
7. 上线推广和运营策略
8. 业务-技术协同机制
9. 可复用的技术组件和方法论
10. 经验教训和最佳实践
11. 待验证问题（需与其他访谈对象交叉验证）
