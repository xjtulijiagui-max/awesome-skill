# 执行层访谈提纲模板

## 访谈对象说明

**典型岗位：**
- 一线员工（客服、销售、运营等）
- 业务执行人员
- 系统操作员
- 基层管理人员

**项目经历要求：**
- 实际使用过AI应用工具或系统
- 对日常工作流程有深入了解
- 有AI工具使用前后的对比体验
- 能够反馈真实的用户体验和问题

**每场时长：** 45-60分钟

**建议人数：** 3-5人，覆盖不同岗位或场景

## 访谈核心目标

从一线使用者的视角了解AI应用的真实效果：
- 了解AI工具在实际工作场景中的使用情况
- 收集用户对AI工具的真实反馈和建议
- 识别AI工具的优点和不足
- 发现工作流程中的实际问题和改进机会
- 提炼用户接受度和使用习惯的影响因素

## 核心访谈问题

### 模块一：工作场景与流程

**问题背景：** 了解访谈对象的日常工作内容和流程

**核心问题：**
1. 您的日常工作是什么？主要包含哪些任务？
2. 工作流程是怎样的？（一步步描述）
3. 哪些任务最耗时？哪些任务最繁琐？
4. 工作中最大的挑战或困难是什么？
5. 工作中需要与哪些人或系统协作？

**预期产出：**
- 工作场景画像
- 工作流程图
- 痛点和难点识别
- 协作关系图

### 模块二：AI工具使用情况

**问题背景：** 了解AI工具的具体使用情况

**核心问题：**
1. 您使用哪些AI工具？分别用于什么场景？
2. 使用频率如何？（每天、每周、偶尔）
3. 如何使用AI工具？（操作流程、步骤）
4. AI工具在您的整个工作流程中处于哪个环节？
5. 使用AI工具前后，工作方式有什么变化？
6. 使用AI工具需要多长时间才能掌握？

**预期产出：**
- AI工具使用清单
- 使用频率统计
- 操作流程记录
- 工作流程对比（前后对比）
- 学习曲线评估

### 模块三：AI工具的价值和效果

**问题背景：** 了解AI工具带来的实际价值和效果

**核心问题：**
1. AI工具帮您解决了什么问题？（具体例子）
2. 使用AI工具后，工作效率有提升吗？提升多少？
3. 工作质量有改善吗？（准确率、满意度等）
4. 最喜欢的功能是什么？为什么？
5. AI工具节省了多少时间？（具体任务的时间对比）
6. 有哪些意外的收获或惊喜？

**预期产出：**
- 价值量化数据（时间、效率、质量）
- 最受欢迎功能清单
- 具体案例（使用场景、效果）
- 意外价值发现

### 模块四：AI工具的问题和不足

**问题背景：** 了解AI工具存在的问题和改进空间

**核心问题：**
1. AI工具有哪些不足之处？
2. 使用过程中遇到的最大困难是什么？
3. 哪些功能不好用？为什么？
4. AI工具的准确性如何？有出错的情况吗？
5. 响应速度如何？能满足工作需求吗？
6. 有哪些功能是AI工具做不到但您很需要的？
7. 有哪些使用场景是AI工具无法覆盖的？

**预期产出：**
- 问题清单（按严重程度排序）
- 不足和限制分析
- 准确性和性能评估
- 功能缺口识别
- 无法覆盖的场景清单

### 模块五：用户接受度和使用习惯

**问题背景：** 了解用户对AI工具的接受程度和使用习惯

**核心问题：**
1. 您对AI工具的整体满意度如何？（打分）
2. 最初使用AI工具时，有顾虑或担心吗？
3. 同事们对AI工具的接受度如何？
4. 是什么让您愿意使用AI工具？
5. 是什么让您不愿意使用或抗拒使用AI工具？
6. 如果不用强制，您会主动使用吗？
7. 有哪些因素会影响您的使用意愿？

**预期产出：**
- 用户满意度评分
- 接受度评估
- 使用驱动力分析
- 阻力因素分析
- 使用意愿影响因素

### 模块六：培训和支持

**问题背景：** 了解AI工具的培训和支持情况

**核心问题：**
1. 您是如何学会使用AI工具的？
2. 培训内容是否充分？是否实用？
3. 培训方式是什么？（线上、线下、文档、视频）
4. 使用过程中遇到问题，如何获得帮助？
5. 有哪些支持资源？（帮助文档、客服、技术支持）
6. 希望获得什么样的培训或支持？

**预期产出：**
- 培训方式和内容评估
- 学习路径和难度评估
- 支持资源清单
- 培训需求分析
- 支持改进建议

### 模块七：改进建议和期望

**问题背景：** 收集用户对AI工具的改进建议

**核心问题：**
1. 如果可以改进AI工具，最希望改进什么？
2. 希望增加哪些新功能？
3. 有哪些使用建议可以分享给其他同事？
4. 对AI工具的未来有什么期望？
5. 如果AI工具能做一件事让您工作更轻松，那会是什么？
6. 有其他想说的吗？

**预期产出：**
- 改进建议清单（优先级排序）
- 新功能需求清单
- 使用技巧和最佳实践
- 用户期望和愿景
- 开放式反馈

## 访谈注意事项

1. **聚焦真实体验：** 关注用户的真实使用感受，避免引导性问题
2. **收集具体案例：** 要求用户举出具体的使用场景和例子
3. **量化数据优先：** 时间、效率、满意度等要尽量量化
4. **保持开放心态：** 用户的反馈可能与预期不同，要接受负面反馈
5. **关注情感因素：** 用户的情感反应（喜欢、讨厌、担忧）很重要

## 输出格式建议

访谈结束后，整理为以下结构：
1. 访谈基本信息
2. 访谈对象画像（岗位、工作内容）
3. AI工具使用情况
4. 价值和效果分析
5. 问题和不足清单
6. 用户接受度评估
7. 培训和支持反馈
8. 改进建议和需求
9. 典型案例（1-2个真实使用场景）
10. 待验证问题（需与其他访谈对象交叉验证）

## 补充说明

执行层访谈的特点：
- **数量要多：** 执行层访谈建议3-5人以上，以获得更全面的反馈
- **形式灵活：** 可以是小组访谈，也可以是个人访谈
- **语言通俗：** 问题要简单易懂，避免专业术语
- **时间控制：** 每场45-60分钟即可，避免疲劳
- **鼓励表达：** 营造轻松的氛围，鼓励用户畅所欲言
