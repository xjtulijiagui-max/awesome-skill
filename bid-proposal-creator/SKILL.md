# 招投标文档生成专家 (Bid Proposal Creator)

## 功能概述

本 Skill 提供**端到端招投标解决方案**，帮助用户快速完成从招标信息搜集到投标文档生成的全流程。

### 核心特性

| 特性 | 说明 |
|------|------|
| 🗺️ **东部交易平台数据库** | 预制东部GDP前10省级公共交易平台地址和API接口 |
| 🔍 **智能标书搜索** | 基于公司名称或平台，自动搜索相似历史标书 |
| 📄 **招标解析** | 自动提取招标要求、资质条件、评分细则 |
| 📝 **初稿生成** | 生成符合规范的Word格式投标文档 |
| 🎯 **行业适配** | 支持福利采购、IT服务、咨询服务等多种类型 |

## 使用方式

### 模式一：浏览器自动化模式（推荐）

直接爬取真实招标网站数据，无需购买 API：

```bash
# 使用 --browser 参数启用浏览器自动化
python3 scripts/bid_creator.py search --company "平安知鸟" --browser --top 10

# 搜索特定关键词
python3 scripts/bid_creator.py search --company "东方福利网" --browser --keywords "福利,工会"
```

**优点：**
- 数据真实，来自采招网等主流平台
- 无需购买第三方 API
- 支持实时搜索

**支持的网站：**
- 采招网 (bidcenter.com.cn) - 已适配
- 中国政府采购网 (ccgp.gov.cn) - 适配中
- 各省公共资源交易中心 - 适配中

### 模式二：模拟数据模式（演示用）

用于演示和测试，无需网络连接：

```bash
# 不使用 --browser 参数，使用内置模拟数据
python3 scripts/bid_creator.py search --company "东方福利网" --top 5
```

## 工具调用

本 Skill 提供以下工具：

### 1. search_bid_documents

搜索相似标书（支持浏览器自动化模式）

**参数：**
- `company_name` (可选): 公司名称
- `platform` (可选): 指定平台名称或URL
- `keywords`: 搜索关键词（如"福利采购"、"工会服务"）
- `province`: 省份限制（如"广东"、"江苏"）
- `date_range`: 时间范围（如"2024-01-01 to 2024-12-31"）
- `use_browser`: 是否使用浏览器自动化（默认 false）

**返回：**
- 相似标书列表（标题、发布时间、招标方、预算金额、相似度、详细链接）

**使用示例：**
```python
# 浏览器自动化模式 - 获取真实数据
searcher = BrowserBidSearcher()
results = searcher.search_by_company_browser("平安知鸟", top_n=10)

# 模拟数据模式 - 用于测试
searcher = BidSearcher()
results = searcher.search_by_company("东方福利网", top_n=5)
```

### 2. parse_bid_document

解析招标文件

**参数：**
- `source`: 招标文件URL或本地文件路径
- `doc_type`: 文件类型（pdf/word/html）

**返回：**
- 结构化招标信息（JSON格式）

### 3. generate_proposal

生成投标初稿

**参数：**
- `bid_info`: 招标信息（parse_bid_document的输出）
- `company_info`: 投标公司信息
- `reference_docs`: 参考标书列表
- `template`: 模板类型（如"福利采购"、"IT服务"）

**返回：**
- Word文档路径
- 生成内容摘要

## 技术实现

### 标书搜索原理

1. **关键词提取**：从公司名称/需求中提取核心业务词
2. **多平台并行搜索**：同时查询多个交易平台
3. **相似度计算**：使用语义相似度模型匹配标书内容
4. **结果排序**：综合相似度、时效性、金额匹配度排序

### 文档生成原理

1. **模板匹配**：根据招标类型选择对应模板
2. **内容填充**：将招标要求映射到模板对应章节
3. **智能生成**：使用大模型生成技术方案、服务承诺等内容
4. **格式渲染**：使用 python-docx 生成标准Word文档

## 故障排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 搜索不到标书 | 平台API限流 | 降低查询频率，使用缓存数据 |
| 标书解析失败 | 文件格式不支持 | 转换为PDF或Word格式后重试 |
| 生成内容质量低 | 参考标书不足 | 提供更多相似标书作为参考 |
| Word格式错乱 | 模板兼容问题 | 使用docx而非doc格式 |
