# Feishu Lightweight KB - 飞书轻量知识库

这是一套无需向量数据库的轻量知识库方案，基于文件目录 + Prompt 驱动检索，专为飞书通道设计。

## 核心特性

- 可选向量检索增强语义能力，支持"向量召回 + 关键词精排"混合检索
- 纯文件目录管理，无需复杂基础设施，向下兼容纯关键词检索
- 多文件连锁搜索，内置同义词扩展
- 优先返回原文片段，并标注来源文件
- 严格遵守"不编造"原则，找不到会明确说明
- 支持多种文档格式：Markdown、TXT、PDF、DOCX
- 按主题分类目录，提升检索效率

## 什么时候使用这个技能

当用户需要：
1. 创建企业内部知识库
2. 对大量文档进行检索问答
3. 基于已有文档回答问题并提供依据
4. 在无法安装向量数据库的环境中搭建轻量知识库
5. 可选向量检索增强，解决语义相似但关键词不匹配问题

## 工作流程

### 第一步：创建知识库目录

按照主题分类创建知识库目录结构：

```bash
mkdir -p ~/kb/company_docs/{hr,finance,training,project,policy}
```

你可以根据实际需求调整分类。

### 第二步：上传文档

引导用户通过飞书通道将文档上传到对应的主题目录。

**推荐文档格式优先级：**
1. Markdown (.md) - 最佳效果
2. 纯文本 (.txt)
3. PDF (.pdf) - 需要系统支持解析
4. Word (.docx) - 需要系统支持解析

**文档命名建议：**
- 使用语义清晰的文件名，如 `报销制度.md`、`新员工入职流程.md`
- 避免使用 `新建文档1.docx`、`最终版(2).pdf` 等无意义名称

**文档结构建议：**
- 使用清晰的标题层级（# 一级标题，## 二级标题）
- 流程和标准使用列表展示
- 这样更利于快速定位相关内容

### 第三步：处理用户提问

当用户提问时，严格按照以下流程执行：

1. **理解问题**：提取核心意图，判断问题类型（制度、流程、标准、定义等）
2. **提取扩展关键词**：不仅使用用户原词，还要扩展同义词、近义词、英文常见词
3. **按目录优先搜索**：根据主题选择相关目录，无法判断则跨目录搜索
4. **多文件连锁搜索**：至少检查 2 个及以上可能相关的文件
5. **定位原文片段**：优先提取最相关的原文，特别关注标题、列表、流程步骤、标准说明、关键数据等
6. **整合并回答**：基于原文整理回答，按指定格式输出

## 必须遵守的规则

1. **严禁编造**：不允许使用知识库之外的信息回答，不允许凭常识补全内容
2. **必须优先返回原文依据**：只要找到相关内容，尽量返回原文片段，不只是总结
3. **必须标注来源**：每次回答都要列出来源文件名和目录
4. **找不到明确依据必须说明**：输出 `知识库中未找到明确相关信息。`
5. **信息不完整必须提示**：输出 `知识库信息不完整，当前仅找到以下相关内容：`
6. **文件冲突必须说明**：明确指出哪些文件存在差异以及差异点，不自行裁决

## 检索增强策略

1. **同义词扩展**：同时搜索用户原始词、扩展词、可能的英文词、相关上下文词
2. **多轮检索**：首轮结果不足时，重新改写关键词，扩展到相邻目录再次搜索
3. **防遗漏**：至少检查 2 个文件，涉及流程标准时优先查找补充文件
4. **结构优先**：优先检索标题、小标题、表格、列表、加粗段落、规则句
5. **长短答案适配**：短答案直接引用，长答案先总结再摘录关键片段

## 输出格式

请按照以下结构化格式输出回答：

【结论】
用简洁语言直接回答用户问题。

【原文片段】
尽量摘录最关键的原文内容。如有多个片段，可分点展示。

【来源】
列出来源文件名，附带目录名。

【说明】
如有以下情况，在这里说明：
- 信息不完整
- 多文件冲突
- 需要用户补充上传文档
- 当前只找到部分相关内容

## 用户引导

如果用户问题比较模糊，先尝试基于可能主题搜索，搜索无结果再提示用户补充更具体的问题。

推荐引导用户使用以下提问方式：
- "根据知识库，公司差旅报销标准是什么？"
- "根据知识库，新员工入职流程分几步？"
- "请基于知识库，找出课程开发规范中的关键步骤，并附原文片段。"

## 向量检索增强（可选）

本技能支持可选的向量检索增强，在纯关键词检索基础上增加语义召回能力，提升召回率。

### 启用方法

1. **安装依赖**
   - 本地方案（推荐，无需 API Key）：`pip install numpy sentence-transformers pypdf python-docx`
   - OpenAI 方案：`pip install openai numpy pypdf python-docx`

2. **构建向量索引**
```bash
python ~/workspace/agent/skills/feishu-lightweight-kb/scripts/build_index.py \
  --kb-dir ~/kb/company_docs \
  --chunk-size 512 \
  --chunk-overlap 50 \
  --model local
```

3. **混合检索流程**
   - 用户提问 → 向量召回 Top 20 相似块 → 关键词精排 → 原文验证 → 整合回答
   - 如果没有索引，自动回退到纯关键词检索
